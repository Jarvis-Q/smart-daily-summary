# 每日工作摘要生成器

## 触发方式与执行逻辑
本 Skill 严格根据用户输入的斜杠命令执行对应分支，**禁止混淆执行**：
 
### 分支 1: `/daily-summary` (仅采集与生成)
1. **数据采集**：运行 `python3 ~/.claude/skills/daily-summary/scripts/gather_data.py` 获取 Git 提交与 Claude Code 对话记录（该脚本仅采集、不发送）。若脚本缺失或非 Git 仓库，优雅降级。
2. **隐性工作补充（强制交互）**：向用户发送以下消息并等待回复：
   > " 基础数据已收集。请快速补充（没有可回复'无'）：
   > 1. ️ **沟通/会议**：重要决议、跨部门沟通或邮件指示？
   > 2.  **卡点/排坑**：技术障碍、环境问题或 Bug 及解决过程？
   > 3.  **知识/资源**：新概念、新工具或关键数据口径？"
3. **生成与保存**：结合数据与用户回复生成标准化 Markdown 日报。**保存路径与文件名严格使用脚本输出的"日报保存位置"那一行**（脚本已按 `local_output_dir` 解析并预建目录），文件名形如 `daily-summary-YYYY-MM-DD.md`——`:send` 的自动定位依赖此命名。
   - **token 与 AI 耗时必须采用脚本输出的真实数值**（脚本已从 jsonl 汇总"真实 token 合计"与"AI 活跃时长合计"），严禁自行估算。当日报拆为多个任务时，各任务的 `Token 消耗量`/`AI 辅助耗时` 之和应等于脚本给出的当日真实合计（按投入比例分摊）。
4. **结束提示**：告知用户日报已保存，提示可通过 `/daily-summary:send` 发送，或在编辑器中修改后再发送。

> ️ **安全护栏（务必遵守）**：日报正文会被发送到私有服务器。本项目为医疗语境，**严禁在日报中写入患者姓名/病历号/手机号/身份证号等个人标识，以及任何密钥、令牌、密码**。只写"做了什么"，不写敏感原文。发送脚本还会在出网前对疑似凭证做二次脱敏兜底。

### 分支 2: `/daily-summary:send` (发送最新日报)
1. 执行命令：`python3 ~/.claude/skills/daily-summary/scripts/send_report.py`（无参时脚本会自动定位 `config.json` 中 `local_output_dir` 下最新的 `daily-summary-YYYY-MM-DD.md` 文件）。
2. 向用户反馈发送结果（成功/失败/解析错误）。

### 分支 3: `/daily-summary:send <文件路径>` (发送指定文件)
1. 接收用户提供的具体文件路径参数。
2. 执行命令：`python3 ~/.claude/skills/daily-summary/scripts/send_report.py <指定文件路径>`（脚本内部会校验文件是否存在，不存在则报错退出）。
3. 向用户反馈发送结果。

## 输出格式规范（仅分支 1 适用）
** 严禁修改标题结构和括号内的字段名，否则会导致 `send_report.py` 解析失败：**
```markdown
### 任务 X: [任务名称] (预估总耗时: X小时)
- AI 辅助耗时: X 小时
- Token 消耗量: X tokens
- 工作详情:
  - 具体工作内容 1
  - 具体工作内容 2
```
说明：
- `AI 辅助耗时` 与 `Token 消耗量` **取自脚本输出的真实数据**（jsonl 汇总），多任务时按投入比例分摊，各任务之和等于当日真实合计；`send_report.py` 用正则 `AI 辅助耗时:\s*([\d.]+)` 与 `Token 消耗量:\s*(\d+)` 抓取数值，请保持数字紧跟字段名。
- `预估总耗时` 为该任务的整体工时（含非 AI 部分），可由你根据工作量合理判断。
- 用户补充的隐性工作（沟通/卡点/知识沉淀）可归纳为独立任务，其 AI 耗时/Token 通常为 0。
