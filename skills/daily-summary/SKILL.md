# 每日工作摘要生成器

## 执行步骤

1. **收集硬数据**：运行 Python 数据收集脚本 `python3 ./scripts/gather_data.py`。
2. **分析对话**：仔细阅读收集到的 Claude Code 对话日志。从中提炼出：
   - 用户的核心 Prompt/指令意图。
   - AI 与用户交互中产生的关键 Insight（如架构决策、技术难点突破、优化思路）。
3. **主动询问软数据**：
   - 在生成最终文件前，**必须**向用户发送以下确认消息：
     > "📊 代码和 AI 对话记录已收集完毕。在生成日报前，请快速补充以下信息（没有可回复'无'）：
     > 1. 🗣️ **沟通/会议**：今天是否有重要的会议决议、跨部门沟通结论或关键邮件指示？
     > 2. 🚧 **卡点/排坑**：今天是否遇到了技术障碍、环境配置问题或 Bug？最终是如何解决的？
     > 3. 💡 **知识/资源**：今天是否学到了新概念、发现了新工具或记录了关键数据口径？"
   - **等待用户回复后**，将回复内容整合进日报。
4. **确认输出路径**：
   - 询问用户："请问您希望将今天的日报保存在哪个目录？（例如：~/Documents/Notes 或当前项目根目录）"
   - 等待用户回复后，使用该路径。
5. **生成并保存文件**：
   - 按下方模板生成内容。
   - 文件名格式必须为：`daily-summary-YYYY-MM-DD.md`。
   - 使用 `mkdir -p` 确保目标目录存在，然后写入文件。

## 输出模板

### 📅 [YYYY-MM-DD] 工作摘要

**🎯 核心产出**
- [基于 Git 提交提炼的 1-3 个关键成果]

**🧠 核心 Insight 与思考**
- [从对话日志中提取的关键技术洞察、架构决策或解决思路]
- [例如：重构了 XX 模块，发现 YY 方案性能更好]

**🗣️ 沟通与协作**
- [用户补充的会议决议、沟通结论]
- [若无，则显示：今日无重要沟通记录]

**🚧 卡点与排坑**
- [用户补充的技术障碍及解决方案]
- [若无，则显示：今日开发顺利，无重大卡点]

**💡 知识沉淀**
- [用户补充的新概念、工具、数据口径]
- [若无，则显示：今日无新增知识沉淀]

**💬 关键指令回顾**
- [列出今天最有价值的 2-3 个 Prompt 或指令方向]

**📝 明日计划/待办**
- [基于今日未完成项、代码 TODO、对话结尾或用户补充推断]
