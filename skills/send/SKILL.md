# 每日工作摘要生成器 · 发送

> 本 skill **只发送，绝不采集或生成**。生成是独立的 `/daily-summary:generate`。
> 严格分离：本 skill 内禁止调用 `gather_data.py`，也不要顺手改写日报内容（除非发送被服务端拒绝且需修正，见下）。

## 用法

> **`<插件根>` 怎么取**：本 skill 加载时，harness 会注入一行
> `Base directory for this skill: <插件根>/skills/send`，**其上两级即插件根**。
> 请据此拼出绝对路径再执行——**不要写死任何绝对路径**：插件通过 marketplace 安装后
> 位于 `~/.claude/plugins/cache/<marketplace>/daily-summary/<版本>/`，因人因版本而异。

**不带参数** —— 发送最新日报：

```bash
python3 <插件根>/scripts/send_report.py
```

脚本会自动定位 `config.json` 中 `local_output_dir` 下最新的 `daily-summary-YYYY-MM-DD.md`。
注意"最新"是按**文件名字典序**取（`sorted(candidates)[-1]`），不是按修改时间——
刚补写的历史日报**不会**被选中，要发它必须显式传路径。

**带参数** —— 发送指定文件：

```bash
python3 <插件根>/scripts/send_report.py <文件路径>
```

脚本内部会校验文件是否存在，不存在则报错退出。`workDate` 由文件名解析，不是当天日期。

## 发送前自检（强烈建议，能省掉一轮失败请求）

服务端对**每个任务**的 `预估总耗时` 有硬性区间要求，越界会整单打回。发送前先本地核对：

- 每个任务的 `预估总耗时` 必须在 **0.5 ~ 4 小时**之间（含两端）
  - 小于 0.5 → `工作耗时最少0.5小时`
  - 大于 4 → `工作耗时不能大于4小时`
- `AI 辅助耗时` 与 `Token 消耗量` 可以为 0，不受此限
- 违规时**不要为了过校验而虚报工时**：零碎工作向上合并、大块工作按真实子结构向下拆分，
  拆分后各任务的 AI 耗时/Token 之和必须仍等于原合计（只重新分配，不增减）

## 执行与结果反馈

运行后向用户反馈发送结果（成功/失败/解析错误）。

**过滤输出时务必保留错误信息**。脚本会打印完整 Payload，输出很长，但失败原因藏在服务端返回的
JSON 里（`"code"` / `"msg"` 行，以引号开头）。用排除式过滤（如 `grep -v '^\s*"'`）会把报错一起滤掉，
导致看不到失败原因而白发一轮。请用白名单式过滤：

```bash
python3 <插件根>/scripts/send_report.py 2>&1 | grep -E '"code"|"msg"|来源|解析到|发送成功|发送失败'
```

## 注意事项

- **非交互即自动确认**：脚本用 `sys.stdin.isatty()` 判断，通过工具调用运行时 stdin 不是终端，
  会**直接发送**、不再二次确认。因此调用前就要确信内容无误——这是一次出网的、不易撤回的动作。
- **失败会自动重试 3 次**（间隔 3 秒）。服务端拒绝时不会产生记录，但重跑一次就是 3 次请求，
  所以定位问题优先靠本地复算，不要靠反复发送试探。
- **重复提交同一 `workDate` 的行为未知**（覆盖还是新增记录不明确）。已成功发送的日报若需更新，
  先与用户确认，不要擅自重发。

## 安全护栏

日报正文会出网。本项目为医疗语境，**严禁**发送含患者姓名/病历号/手机号/身份证号等个人标识的内容，
以及任何密钥、令牌、密码。脚本在出网前会对疑似凭证做保守脱敏兜底（命中时会打印脱敏处数），
但那是兜底，不是许可——内容本身就不该包含敏感原文。
